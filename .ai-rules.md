# Portfolio Tracker Development Principles

## Architecture
- Backend is stateless, exposing a pure REST API.
- All domain logic, currency calculations, and financial metrics happen strictly in `apps/api`.
- Frontends maintain zero business logic; they only handle state presentation and UI components.

## TypeScript & Node.js Standards
- Use strict typing; avoid `any`.
- Do not comment the code
- Use async/await with clean try/catch error-handling blocks.
- SOLID Principles
- Clean Code Principles
- KISS (Keep It Simple, Stupid)
- YAGNI (You Ain't Gonna Need It)
- DRY (Don't Repeat Yourself)
- Composition over Inheritance
- Separation of Concerns (SoC)
- Test-Driven Development (TDD)
- Behavior-Driven Development (BDD)
- Consistent Naming Conventions
- Automated Code Formatting and Linting
- Use of established Design Patterns
- Meaningful Version Control Commits
- Modular and Componentized Architecture for Reusability
- Continuous Refactoring after code generation
- Single Responsibility Principle (SRP) for functions, classes, and components
- Tell Don’t ask.
